# Homepage Scope and Template Page Plan

## Goal
Keep the strongest sections needed for the first real DreamPlay homepage pass, and move the less-important reference-clone sections into a separate template page so we preserve the work without letting the homepage stay overcrowded.

## Current homepage section inventory
Current `/` is exporting the animated `home2` page, which currently includes these major sections in order:

1. Hero slider
2. Brand intro
3. Category cards
4. Video section
5. Featured product
6. Comparison section
7. Bundle builder
8. Marquee
9. Social feed
10. Countdown timer / sale section
11. Product grid
12. Testimonial
13. Tabbed categories
14. Blog / journal section
15. Value props
16. Footer / popup / back to top

## Core decision
For the first homepage pass, we should make the homepage shorter, clearer, and more product-directed.

I agree with Lionel’s direction:
- do NOT delete the extra sections
- do NOT throw away the reference-fidelity work
- instead move the non-essential sections into a template page

That gives us:
- a cleaner real homepage
- preserved concept/reference work
- a reusable showroom page for future sections, A/B ideas, or later reintroduction

---

## Recommended homepage-first scope
These are the sections I recommend keeping on the real homepage for the first pass.

### Keep on homepage
1. Hero
- reason: first impression, flagship CTA, product identity

2. Brand intro
- reason: quick positioning and transition from aesthetic to story

3. Featured product
- reason: homepage should clearly sell a flagship product

4. Bundle builder OR a simplified bundle/promo module
- reason: strong commercial intent and useful bridge into shopping behavior
- note: we may simplify this later, but it deserves to stay in the homepage-first scope

5. One supporting lifestyle / ecosystem section
- best candidate:
  - Video section OR app/connectivity section depending on real DreamPlay asset mapping
- reason: homepage needs one emotional/feature-rich section beyond pure product merch

6. Testimonial / trust proof
- reason: helps support conversion and brand credibility

7. Value props
- reason: good closing utility/trust layer before footer

### Optional keep
8. One compact category/shop-navigation module
- this could be preserved in a reduced form if needed for homepage-to-shop navigation
- but it should be simplified, not left as a large theme-demo section

---

## Recommended move to template page
These are the sections I recommend moving off the homepage into a template page.

### Move off homepage
1. Full category card block
- useful as a reference/theme section
- not essential for the first DreamPlay homepage pass

2. Comparison section
- strong section, but not homepage-critical in the very first pass
- can be reintroduced later when product comparison logic is clearer

3. Marquee
- visually nice, but not core to the first business-focused homepage

4. Social feed
- currently more of a theme/demo fidelity section than a must-have commerce homepage section

5. Countdown timer / sale block
- good as a promo template section, but not always needed on the primary homepage

6. Product grid
- the homepage should not feel like a giant catalog page if `/shop` is its own destination

7. Tabbed categories
- useful reference section, but not part of the first simplified homepage pass

8. Blog/journal block
- not urgent for homepage round one
- can live on the template page or be reintroduced later in a much lighter form

---

## Proposed template page purpose
Create a page whose job is to preserve the cloned reference modules.

### Suggested purpose
A “template showroom” page that holds the non-essential but still valuable sections.

### Good uses for this page
- preserve Shopify Concept-inspired section work
- reuse sections later without rebuilding them
- compare alternate homepage structures
- keep a living library of proven merch/content blocks

### Suggested route names
Preferred options:
1. `/home-template`
2. `/template-showroom`
3. `/section-library`

My recommendation:
- `/home-template`

Reason:
- clear
- low-friction
- obviously connected to the homepage work
- easy for us to remember

---

## Recommended split

### Homepage should become
A shorter high-conversion DreamPlay storefront homepage focused on:
- flagship product
- brand story
- real DreamPlay assets
- one commerce interaction block
- one ecosystem/lifestyle block
- trust layer

### Template page should become
A preserved reference page containing:
- category cards
- comparison section
- marquee
- social feed
- countdown section
- product grid
- tabbed categories
- blog block
- any other non-essential but well-executed theme sections

## Implementation sequence

### Phase 1
Create the homepage/template split plan and lock the section list.

### Phase 2
Create a new template page route and move the non-essential homepage sections there.

### Phase 3
Leave the homepage with only the kept sections, still using the current components for now.

### Phase 4
Do the homepage first-pass DreamPlay replacement work:
- real DreamPlay hero/product logic
- real asset swaps
- stronger DreamPlay merchandising narrative

### Phase 5
Later do the shop page pass.
